# Стратегия монетизации iOS-приложений (Brite Technologies)

---

## Философия монетизации

**Принцип**: Пользователь должен почувствовать ценность продукта _до_ покупки, но _быстро_ упереться в лимиты, чтобы конвертироваться.

**Двухэтапный релиз в App Store:**
> 1. **R1**: Чистое приложение без монетизации → проходим ревью Apple
> 2. **R2**: Добавляем paywall, подписки, premium-фичи
>
> Это де-рискует процесс: Apple не отклонит из-за проблем с подписками в R1.

---

## Модель подписок

### Стандартная линейка (применять всегда):

| Продукт | Product ID | Цена | Триал |
|---------|-----------|------|-------|
| Weekly | `{BUNDLE_ID}.weekly` | ~$4.99/нед | Нет |
| Yearly | `{BUNDLE_ID}.yearly` | ~$39.99/год | 3 дня |

**Правила ценообразования:**
- Yearly всегда выбран по умолчанию (preselected)
- Yearly показывается с "3-day free trial"
- Weekly без триала — как быстрый вход для импульсивных покупателей
- Соотношение yearly/weekly ≈ 8× (мотивирует yearly)

### Дополнительные продукты (опционально):
- Super Pro: отдельная группа подписок → два одновременных триала → выше LTV
- Welcome offer (3 месяца по сниженной цене)

## Уровни жёсткости монетизации

### Минимальный (freeItemLimit = 3)
**Когда применять**: Новые рынки, высококонкурентные ниши, приложения с долгим циклом вовлечения

```
Бесплатно: 3 объекта/записи
Платно: всё остальное
```

Пример: трекер упражнений → 3 тренировки бесплатно

### Стандартный (freeItemLimit = 1) ← РЕКОМЕНДУЕТСЯ
**Когда применять**: Большинство утилит, трекеров, органайзеров

```
Бесплатно: 1 объект/запись
Платно: все последующие + premium-фичи
```

Пример: GLP-1 Tracker → 1 инъекция бесплатно, всё остальное за подписку

### Максимальный (freeItemLimit = 0)
**Когда применять**: Premium-ниши, узкоспециализированные инструменты

```
Бесплатно: только просмотр (демо-данные или пустой экран)
Платно: любое взаимодействие с контентом
```

Пример: сканер антиквариата → просмотр без сканирования

## Архитектура FeatureGate

```swift
enum FeatureGate {
    // Базовый лимит
    static let freeItemLimit: Int  // 0, 1 или 3

    // Добавление нового объекта
    static func canAddItem(currentCount: Int, isPro: Bool) -> Bool {
        isPro || currentCount < freeItemLimit
    }

    // Premium-фичи (всегда только платно):
    static func canExportPDF(isPro: Bool) -> Bool { isPro }
    static func canAccessAnalytics(isPro: Bool) -> Bool { isPro }
    static func canUseAI(isPro: Bool) -> Bool { isPro }
    static func canSyncCloud(isPro: Bool) -> Bool { isPro }

    // Адаптировать под приложение:
    // canScanItem, canAccessHistory, canShareData, etc.
}
```

## Что блокировать (стандартный набор)

| Фича | Free | Premium |
|------|------|---------|
| Создание объектов | ≤1 (или ≤3) | Неограниченно |
| Экспорт в PDF | ❌ | ✅ |
| Аналитика / графики | ❌ | ✅ |
| AI-функции | ❌ | ✅ |
| Синхронизация iCloud | ❌ | ✅ |
| Уведомления / ремайндеры | ❌ | ✅ |
| Дополнительные вкладки | ❌ | ✅ |
| Экспорт / шаринг | ❌ | ✅ |

## Правила UX блокировок

### ✅ Правильно:
1. **Frosted glass overlay** (`.ultraThinMaterial`) на заблокированной секции — тап открывает пейвол
2. **PremiumBannerView** внизу списка — видимый, но ненавязчивый
3. **Кнопка + блокируется** — тап открывает пейвол (не скрывать кнопку!)
4. **Полный overlay вкладки** — если вся вкладка premium-only

### ❌ Неправильно:
- Показывать текст "Premium Feature 🔒" на карточках
- Скрывать заблокированные кнопки (пользователь должен видеть что есть)
- Показывать пейвол слишком часто (раздражает)
- Показывать пейвол только из Settings

## Remote Config: задержка крестика

**Ключ**: `{APP_KEY}_close_button_delay`
**Тип**: Number (Double)

| Значение | Поведение |
|---------|-----------|
| `0` | Крестик сразу |
| `5` | Крестик через 5 секунд *(дефолт)* |
| `10` | Крестик через 10 секунд (агрессивно) |
| `-1` | Крестик никогда (хардпейвол) |

**Важно**: Крестик НИКОГДА не должен быть виден сразу на production.
Только через Remote Config delay. Это позволяет A/B тестировать без обновления приложения.

## Apple Review — требования к монетизации

Для прохождения ревью:

- ✅ Кнопки ToU и PP на каждом paywall экране
- ✅ Restore Purchases кнопка видима
- ✅ Подписки в App Store Connect созданы до подачи на ревью
- ✅ Все premium-фичи доступны при активной подписке
- ✅ Trialing явно указан (3-day free trial)
- ❌ Нет обязательной подписки для базовых функций (хотя бы 1 объект бесплатно)
- ❌ Нет exit-intent задержек (>0.1 сек на крестике — rejection 5.6)
- ❌ Нет ложных "SALE" / "LIMITED TIME" без реальных условий

## Настройка подписок в App Store Connect

1. Создать группу подписок: `{APP_DISPLAY_NAME} Premium`
2. Добавить yearly: `{BUNDLE_ID}.yearly`
   - Reference name: "Yearly"
   - Price: Tier 8 (~$39.99)
   - Introductory offer: Free Trial 3 days
3. Добавить weekly: `{BUNDLE_ID}.weekly`
   - Reference name: "Weekly"
   - Price: Tier 5 (~$4.99)
   - No introductory offer

## Аналитика конверсии

Обязательные события для отслеживания:

```
{APP_KEY}_paywall_shown        → rate shown / sessions
{APP_KEY}_purchase_started     → intent rate
{APP_KEY}_purchase_completed   → conversion rate
{APP_KEY}_purchase_cancelled   → drop-off
{APP_KEY}_paywall_dismissed    → dismiss rate (зависит от delay)
{APP_KEY}_restore_purchases    → returners
```

**Целевые показатели (App Store average):**
- Paywall → Trial start: 3–8%
- Trial → Paid: 40–60%
- Overall: 1–4% DAU → paid
